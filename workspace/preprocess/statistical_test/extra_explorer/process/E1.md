# E1 — marginal harmonization (train → validator)

**Intent:** After the usual **robust** transform (`transform_meta.json`), apply a **second stage** fit **only on labeled train** that pushes each feature’s **marginals** closer between **train** and **validator** traffic (e.g. per-feature **quantile / rank-Gaussian** mapping, or another train-fitted monotone map). Then **retrain** LGBM on the harmonized robust tables and re-measure **shift + task** vs **E0**.

**Why:** E0 measures raw train vs validator shift on robust features. E1 tries to **reduce KS / PSI** on those features without a full new feature set — so the miner’s model sees distributions closer to validator logs.

**Important:** The extra_explorer README marks this as **“when implemented”**. This repo does **not** yet ship a single `harmonize_*.py` that writes harmonized parquets + a sidecar meta. Until that exists, E1 is the **spec**; you either add that step locally or skip to **E2** (stronger robust knobs: `--q-low` / `--scaled-clip-abs`, refit meta, retrain) which needs no new harmonization code.

---

## What you must produce before the commands below

1. Harmonized **train** / **val** parquets (same columns as E0 robust, same `label` semantics), e.g. under  
   `workspace/dataset/robusted_dataset/train/E1_harmonized/` (or your choice).
2. Harmonized **test** parquets (`holdout_*`, `pb_human_bot*`) with the **same** mapping as train.
3. Harmonized **validator** log table used for shift: same file role as E0’s  
   `workspace/ssl_data/raw_data/miner_1/validator_request_robusted.parquet`, but passed through the **same** harmonization apply step.
4. A **new** LGBM artifact directory, e.g. `workspace/model/artifacts/lgbm_E1_harmonized/` with `lgbm_classifier.joblib` and `metrics.json`.

Adjust paths in the commands to match your real artifact names.

## Shift (train vs harmonized validator)

Same script as E0; point `--train-parquet` at **harmonized train** and `--validator-parquet` at **harmonized validator** request.

## Task metrics (cross-eval)

Mirror E0: same eval parquets **after** harmonization, new model + its `metrics.json`. Primary table threshold **0.5** matches common `cross_dataset_eval` defaults; `*_at_selected` still comes from `metrics.json` when you omit `--selected-threshold`.

```bash
PYTHONPATH=. python workspace/test/cross_dataset_eval.py \
  --model workspace/model/artifacts/lgbm_E1_harmonized/lgbm_classifier.joblib \
  --metrics-json workspace/model/artifacts/lgbm_E1_harmonized/metrics.json \
  --threshold 0.5 \
  --eval-parquet workspace/dataset/robusted_dataset/test/E1_harmonized/holdout_train.parquet \
  --eval-parquet workspace/dataset/robusted_dataset/test/E1_harmonized/holdout_test.parquet \
  --eval-parquet workspace/dataset/robusted_dataset/test/E1_harmonized/pb_human_bot.parquet \

  --out-dir workspace/preprocess/statistical_test/extra_explorer/runs/E1_harmonization_v1/cross_eval
```
