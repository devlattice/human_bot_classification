# E2 — stronger robust preprocessing (refit meta + retrain)

**Intent:** Change **only** the robust stage (`robust_feature_transform.py`): slightly **tighter** quantile clipping and/or **tighter** post–robust-scale clipping so tails are less extreme before LGBM. **Refit** `transform_meta.json` on **train**, **re-apply** the new meta to val / all eval parquets / validator request, **retrain** a new LGBM, then re-run **shift + cross-eval + collect** vs **E0**.

**Why:** E1 needs extra harmonization code. E2 uses tooling you already have. Stronger clipping often **reduces train vs validator KS** on heavy-tailed features; the risk is **task** regression if you clip away real class signal — that is why `EXPERIMENT_SHEET.md` gates on both shift and holdout metrics.

**Contrast vs E0 (example numbers — tune one axis at a time):**

| Knob | Typical E0-style (baseline) | E2 candidate |
|------|-----------------------------|----------------|
| `--q-low` / `--q-high` | `0.01` / `0.99` | `0.02` / `0.98` |
| `--scaled-clip-abs` (with `--enable-robust-scale`) | `8.0` | `6.0` |

Keep `--keep-features-file`, `--restrict-to-keep-features`, `--enable-log1p`, and `--fit-stats-from` aligned with E0 unless you explicitly change feature policy (that would be a different experiment).

## 1) Refit robust train/val (new `out-dir`, new meta)

Point `--data-dir` at the **same** unpreprocessed train directory you used for E0 (must contain `train.parquet` + `val.parquet`). Use a **new** `--out-dir` so E0 artifacts stay untouched.

```bash
cd /home/dr/Workspace/Poker44-subnet

PYTHONPATH=. python workspace/preprocess/robust_feature_transform.py \
  --data-dir workspace/dataset/unpreprocessed/train \
  --out-dir workspace/dataset/robusted_dataset/train_E2_stronger_robust \
  --fit-stats-from workspace/dataset/unpreprocessed/train/train.parquet \
  --keep-features-file workspace/preprocess/feature_selection/keep_features.txt \
  --restrict-to-keep-features \
  --q-low 0.02 --q-high 0.98 \
  --enable-log1p --enable-robust-scale --scaled-clip-abs 6.0 \
  --drop-row-nan-frac-over -1
```

This writes `transform_meta.json` under `train_E2_stronger_robust/`. All downstream applies for this run must use **this** meta, not E0’s.

## Discipline

- Change **only** robust knobs (or one knob per run) so you know what moved.
- Never mix **E0** `transform_meta.json` with **E2** train rows or the reverse.
- If you only want to see transfer of the **old** model on E2 features, that is a diagnostic — not a substitute for step 3; the proper E2 comparison is **retrained** LGBM on E2 data.
