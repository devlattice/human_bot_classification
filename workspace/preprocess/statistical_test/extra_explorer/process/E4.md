# E4 — small surgical feature ablation (2–5 drops)



---

## 1) Choose drops (example workflow)

1. Sort E0 (or E2) `train_vs_validator_shift.csv` by `ks_statistic` descending.
2. Cross-check candidates with your feature stats: `workspace/preprocess/statistical_test/train_anova.csv`, permutation / gain importance from a baseline model, or leave-one-feature-out runs you already logged.
3. Write a **new** keep list, e.g. `workspace/preprocess/statistical_test/artifacts/feature_selection_m_2/keep_features_E4.txt` — same as the parent `keep_features.txt` but **without** the chosen names (one name per line, no typos).

Keep the same robust **knobs** as E0 (or as E2 if E4 is meant to stack on E2 — then use E2’s `q-low` / `scaled-clip-abs` and document that in `--notes`).

---

## 2) Refit robust train/val with the shorter keep list

```bash
cd /home/dr/Workspace/Poker44-subnet

PYTHONPATH=. python workspace/preprocess/robust_feature_transform.py \
  --data-dir workspace/dataset/unpreprocessed/train/system_human_bot \
  --out-dir workspace/dataset/robusted_dataset/train/train_E4_ablation \
  --fit-stats-from workspace/dataset/unpreprocessed/train/system_human_bot/train.parquet \
  --keep-features-file workspace/preprocess/statistical_test/artifacts/feature_selection_m_2/keep_features_E4.txt \
  --restrict-to-keep-features \
  --q-low 0.01 --q-high 0.99 \
  --enable-log1p --enable-robust-scale --scaled-clip-abs 8.0 \
  --drop-row-nan-frac-over -1
```

`transform_meta.json` is written under `workspace/dataset/robusted_dataset/train/train_E4_ablation/`. Adjust `--q-low` / `--q-high` / `--scaled-clip-abs` to match the parent run (E0 or E2) you are ablating from.

---

## 3) Apply-only: eval tables + validator request

Same pattern as E2: for **each** unpreprocessed eval parquet, apply **E4** meta only.

```bash
META=workspace/dataset/robusted_dataset/train/train_E4_ablation/transform_meta.json

PYTHONPATH=. python workspace/preprocess/robust_feature_transform.py \
  --transform-meta-in "$META" \
  --in-parquet workspace/dataset/unpreprocessed/test/holdout_train.parquet \
  --out-parquet workspace/dataset/robusted_dataset/test/test_E4_ablation/holdout_train.parquet
```

Repeat for `holdout_test`, `pb_human_bot`, `pb_human_bot_val`, and rebuild the **robusted validator** file used for shift (E4 meta throughout). If your unpreprocessed eval tables live under `unpreprocessed/test/system_human_bot/`, point `--in-parquet` there instead.

Fill `<N>` and `<names>` with what you actually removed.

## Pass / fail vs E0

Use `../EXPERIMENT_SHEET.md`: shift should improve or at least not worsen badly on **max_ks** / **count_ks_ge_0_20**, and task metrics must stay within tolerances vs E0. If AUC or recall drops, **revert** the drop list even if KS looked better.
