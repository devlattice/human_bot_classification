# E3 — small surgical feature ablation (2–5 drops)

**Intent:** Remove **only a few** features that are **both** (a) **high train vs validator shift** (e.g. top KS on `train_vs_validator_shift.csv`) **and** (b) **weak for the task** (low importance, ablation win, or flat ANOVA — use evidence you already trust). Then **refit** robust preprocessing with a **new** `keep_features*.txt` (or equivalent list), **re-apply** to val / all eval parquets / validator request, **retrain** LGBM, and re-run **shift + cross-eval + collect** vs **E0**.

**Why:** E2 changes global tails; E3 removes dimensions that hurt domain alignment **if** they do not pay for themselves on holdout. This is **not** “optimize KS only” — every drop must be justified by **task-neutral or task-positive** evidence, then confirmed on `EXPERIMENT_SHEET.md` gates.

**Discipline**

- Drop **2–5** features per run, not half the set.
- Prefer features that are **redundant** with kept columns (correlated, same story as another column) when shift is high.
- **One experiment per ablation list** — if you drop A+B in one run and metrics move, you still do not know which drop did what unless you also ran singles (optional follow-up).

---

## 1) Choose drops (example workflow)

1. Sort E0 (or E2) `train_vs_validator_shift.csv` by `ks_statistic` descending.
2. Cross-check candidates with your feature stats: `workspace/preprocess/statistical_test/train_anova.csv`, permutation / gain importance from a baseline model, or leave-one-feature-out runs you already logged.
3. Write a **new** keep list, e.g. `workspace/preprocess/statistical_test/artifacts/feature_selection_m_2/keep_features_E3.txt` — same as the parent `keep_features.txt` but **without** the chosen names (one name per line, no typos).

Keep the same robust **knobs** as E0 (or as E2 if E3 is meant to stack on E2 — then use E2’s `q-low` / `scaled-clip-abs` and document that in `--notes`).

---

## 2) Refit robust train/val with the shorter keep list

```bash
cd /home/dr/Workspace/Poker44-subnet

PYTHONPATH=. python workspace/preprocess/robust_feature_transform.py \
  --data-dir workspace/dataset/unpreprocessed/train/system_human_bot \
  --out-dir workspace/dataset/robusted_dataset/train/train_E3_ablation \
  --fit-stats-from workspace/dataset/unpreprocessed/train/system_human_bot/train.parquet \
  --keep-features-file workspace/preprocess/statistical_test/artifacts/feature_selection_m_2/keep_features_E3.txt \
  --restrict-to-keep-features \
  --q-low 0.01 --q-high 0.99 \
  --enable-log1p --enable-robust-scale --scaled-clip-abs 8.0 \
  --drop-row-nan-frac-over -1
```

`transform_meta.json` is written under `workspace/dataset/robusted_dataset/train/train_E3_ablation/`. Adjust `--q-low` / `--q-high` / `--scaled-clip-abs` to match the parent run (E0 or E2) you are ablating from.

## 3) Apply-only: eval tables + validator request

Same pattern as E2: for **each** unpreprocessed eval parquet, apply **E3** meta only.

```bash
META=workspace/dataset/robusted_dataset/train/train_E3_ablation/transform_meta.json

PYTHONPATH=. python workspace/preprocess/robust_feature_transform.py \
  --transform-meta-in "$META" \
  --in-parquet workspace/dataset/unpreprocessed/test/holdout_train.parquet \
  --out-parquet workspace/dataset/robusted_dataset/test/test_E3_ablation/holdout_train.parquet
```

Repeat for `holdout_test`, `pb_human_bot`, `pb_human_bot_val`, and rebuild the **robusted validator** file used for shift (E3 meta throughout). If your unpreprocessed eval tables live under `unpreprocessed/test/system_human_bot/`, point `--in-parquet` there instead.
